## Single Page Application (SPA)

**Single Page Application (SPA)** é um tipo de aplicação web que carrega **uma única página HTML** e atualiza dinamicamente o conteúdo conforme o usuário interage com a interface, sem recarregar a página inteira.

### Características principais
- **Navegação rápida:** apenas partes da interface são atualizadas.
- **Menos requisições completas ao servidor:** comunicação geralmente via **APIs** (REST ou GraphQL).
- **Experiência semelhante a aplicativos desktop/mobile.**
- **Uso intenso de JavaScript** no lado do cliente.

### Exemplos de tecnologias usadas
- React
- Angular
- Vue.js

### Vantagens
- Melhor experiência do usuário
- Interface mais rápida após o carregamento inicial
- Separação clara entre frontend e backend

### Desvantagens
- Carregamento inicial mais pesado
- SEO pode exigir estratégias adicionais
- Maior complexidade no gerenciamento de estado e rotas

## Framework vs Library

**Library** é um conjunto de funções ou ferramentas que o desenvolvedor **pode usar quando quiser** dentro do seu código. O controle da aplicação continua sendo do desenvolvedor.

**Framework** é uma estrutura que **define a arquitetura e o fluxo da aplicação**. O desenvolvedor escreve o código seguindo as regras e padrões definidos pelo framework.

### Diferença principal
- **Library:** você chama a biblioteca quando precisa.
- **Framework:** o framework chama seu código (inversão de controle).

### Exemplos
- **Libraries:** React, Lodash, jQuery  
- **Frameworks:** Angular, Django, Ruby on Rails
